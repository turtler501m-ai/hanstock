# 전략탭 구현 진행 현황

작성일: 2026-06-01

## 1. 목표

전략탭 재설계안 기준으로 실제 구현률을 95% 수준까지 끌어올리는 것이 현재 목표다.

95% 완료 기준은 단순 UI 추가가 아니라 다음 조건을 만족하는 상태로 본다.

- AI 전략을 생성, 수정, 선택, 검증, 승인, 폐기할 수 있다.
- 선택된 AI 전략이 후보 탐색, 랭킹, 스코어 저장에 실제로 반영된다.
- 후보 탐색 결과에 어떤 전략과 모델이 사용됐는지 추적 가능하다.
- 전략 승인 전에 정적 검증, API 검증, 백테스트, 페이퍼 검증 결과를 확인할 수 있다.
- 전략탭 UI에서 현재 active 전략, 안전 설정, 검증 상태, 최근 사용 상태를 볼 수 있다.
- 핵심 API와 저장소 동작이 테스트로 보호된다.

## 5. 현재 미완료/취약 영역

### 5.1 백테스트 결과가 승인 게이트에 충분히 연결되지 않음

현재 전략에는 `last_backtested_at` 컬럼이 있으나, 백테스트 리포트 자체를 충분히 저장하고 승인 API가 이를 엄격히 확인하는 구조는 아직 부족하다.

필요한 보강:

- 백테스트 실행 API 추가
- 백테스트 결과 저장
- 수익률보다 MDD, 거래 수, 승률, 손익비, 수수료/슬리피지 반영 여부 중심 평가
- 승인 API에서 백테스트 통과 여부 확인

### 5.2 페이퍼 검증 루프 부족

현재 `last_paper_started_at`, `last_paper_completed_at` 컬럼은 있으나 실제 전략별 모의운용 결과와 승인 조건이 아직 약하다.

필요한 보강:

- 페이퍼 검증 시작/완료 API
- 최소 운용일 또는 최소 후보 생성 횟수 기준
- 페이퍼 결과 저장
- 승인 전 페이퍼 검증 요구 여부 설정

### 5.3 전략 상태 전이 규칙이 아직 느슨함

현재는 사용자가 승인 버튼을 누르면 전략이 approved가 될 수 있다.

목표 상태에서는 다음 순서를 강제해야 한다.

```text
draft -> static_verified -> verified -> backtested -> paper_running -> paper_passed -> approved -> retired
```

단, 내장 룰 전략은 예외 정책을 둘 수 있다.

### 5.4 UI의 검증 산출물 표시 부족

전략 목록에서 상태와 버튼은 볼 수 있지만, 검증 상세 결과와 이벤트 타임라인을 보기에는 아직 부족하다.

필요한 보강:

- 최근 검증 결과 요약 표시
- 백테스트 결과 요약 표시
- 페이퍼 결과 요약 표시
- 이벤트 타임라인 또는 상세 모달

### 5.5 성과 비교/자동 승격/강등 미구현

현재 후보 생성 기록은 남지만 전략별 성과 비교까지 이어지지는 않는다.

필요한 보강:

- 전략별 후보 적중률
- 후보 생성 후 N일 수익률 추적
- 전략별 MDD/승률/거래 수 비교
- 성과 악화 시 자동 retired 또는 review_required 상태 전환

## 8. 2026-06-01 추가 구현 반영

전략 생명주기 중 가장 약했던 백테스트/페이퍼 검증과 승인 게이트를 1차 구현했다.

### 8.1 추가된 API

다음 API가 추가됐다.

- `GET /api/strategy-context`
- `POST /api/ai-strategies/{id}/backtest`
- `POST /api/ai-strategies/{id}/paper/start`
- `POST /api/ai-strategies/{id}/paper/complete`

`/api/strategy-context`는 현재 active 전략, 안전 설정, fallback 상태, 승인 게이트 상태를 반환한다.

### 8.2 승인 게이트 강화

`POST /api/ai-strategies/{id}/approve`는 이제 바로 승인하지 않고 다음 조건을 확인한다.

- 정적 검증 통과
- OpenAI 전략인 경우 API 검증 통과
- `AI_REQUIRE_BACKTEST_PASS=true`인 경우 백테스트 통과
- profile의 `paper_trading_required_days`가 1 이상이면 페이퍼 검증 통과

조건이 부족하면 `409 Conflict`로 승인을 막고 누락된 검증 항목을 반환한다.

### 8.3 상태 전이 보강

전략 상태값이 다음 단계까지 확장됐다.

```text
draft -> verified -> backtested -> paper_running -> paper_passed -> approved
```

검증 실패 또는 재검토가 필요한 경우 `review_required` 상태를 사용한다. 폐기된 전략은 `retired` 상태가 된다.

### 8.4 UI 반영

전략 목록 액션에 다음 버튼이 추가됐다.

- 백테스트
- 페이퍼시작
- 페이퍼완료

## 10. 2026-06-01 95% 근접 보강

전략 운영 감사가 단순 후보 점수 집계에서 실제 운영 추적에 더 가깝게 확장되었다.

- `scanned_candidates`에 `forward_return_1d`, `forward_return_5d`, `forward_return_20d`, `return_updated_at` 컬럼을 추가했다.
- `daily_charts` 캐시를 기준으로 후보 포착일 이후 1/5/20일 사후수익률을 계산하는 `refresh_scanned_candidate_forward_returns()`를 추가했다.
- 전략 성과 API가 평균 사후수익률, 승률, 수익률 샘플 수를 반환한다.
- 승인 대기 주문과 거래 기록에 `strategy_id`, `strategy_version`, `profile_hash`, `source_candidate_id/source_approval_id`를 저장한다.
- 전략 성과 API가 전략별 주문 수, 체결 수, 체결률, 주문 상태 분포를 함께 반환한다.
- 전략 감사 UI의 마지막 카드가 최근 5일 후보 성과와 체결률을 표시한다.
- 성과 재검토 로직이 낮은 평균 점수, 높은 fallback 비율뿐 아니라 음수 5일 후보 수익률과 낮은 체결률도 경고에 포함한다.
- 대시보드 route/core 분리 이후에도 테스트와 운영 주입값이 같은 경로를 보도록 public override 호환성을 보강했다.
- QuantConnect MNQ와 선물 신호 테스트에서 외부 상태나 stale env가 결과를 오염시키지 않도록 서비스/credential/cache 조회 경로를 정리했다.
