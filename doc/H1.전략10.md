> 뉴스와 시장 내러티브의 **강도 · 방향 · 가속도**를 점수화하고,
>
>
> 해당 테마에 연결된 종목으로 점수를 분배해 단기 모멘텀 후보를 찾는 전략입니다.
>

---

## 1. 핵심 요약

**Narrative Momentum**은 시장이 지금 어떤 이야기에 돈을 걸고 있는지를 종목 시그널로 변환하는 전략입니다.

예를 들어, 시장에서 며칠 연속으로 “AI 반도체 슈퍼사이클” 이야기가 강해지고 있고,

그 내러티브가 긍정적이며, 관련 종목 매핑에 삼성전자와 SK하이닉스가 있다면

이 종목들에게 내러티브 점수를 부여합니다.

> ✅ 핵심 가정
>
>
> **강해지고 있고, 긍정적이며, 며칠 연속 부각되는 테마일수록해당 테마 종목이 단기 모멘텀을 받을 가능성이 높다.**
>

---

## 2. 전체 흐름

```
매일 생성되는 내러티브 분석
        ↓
테마별 점수 계산
        ↓
테마 → 종목 매핑
        ↓
종목별 점수 누적
        ↓
최종 종목 시그널 생성
```

| 단계 | 설명 |
| --- | --- |
| 1 | 매일 뉴스 기반 내러티브 JSON 생성 |
| 2 | 테마의 강도, 센티먼트, 방향, 가속도 계산 |
| 3 | 테마명을 종목 매핑 키와 연결 |
| 4 | 연결된 종목에 내러티브 점수 분배 |
| 5 | 종목별 composite score 산출 |
| 6 | 상위 후보 종목 저장 및 화면 표시 |

---

## 3. 필요한 입력 데이터

이 전략을 실행하려면 크게 2가지 입력이 필요합니다.

| 입력 | 설명 |
| --- | --- |
| `narrative_history` | 최근 3일간의 내러티브 분석 데이터 |
| `theme_map` | 테마명과 종목을 연결한 매핑 데이터 |

---

# 4. 입력 1 — narrative_history

매일 1개씩 쌓이는 JSON 데이터입니다.

전략은 이 중 **최근 3일치**를 사용합니다.

```
{
  "date":"2026-02-23",
  "market_mood":"risk_on",
  "mood_score":72,
  "dominant_narratives": [
    {
      "theme":"AI 및 반도체 슈퍼사이클 지속",
      "strength":90,
      "sentiment":"bullish",
      "direction":"rising",
      "affected_sectors": ["tech","semiconductor"],
      "key_facts": ["...","..."]
    }
  ],
  "narrative_shifts": [
    {
      "theme":"방산",
      "change":"new"
    }
  ]
}
```

---

## narrative_history 주요 필드

| 필드 | 의미 |
| --- | --- |
| `date` | 내러티브 분석 날짜 |
| `market_mood` | 시장 분위기 |
| `mood_score` | 시장 분위기 점수 |
| `dominant_narratives` | 그날 시장에서 부각된 주요 내러티브 |
| `theme` | 테마명 |
| `strength` | 테마 강도, 0~100 |
| `sentiment` | bullish, positive, neutral, mixed, negative, bearish |
| `direction` | rising, stable, fading |
| `affected_sectors` | 영향을 받는 섹터 |
| `narrative_shifts` | 새로 등장하거나 약해진 테마 변화 |

---

## 사용 기준

이 전략은 모든 내러티브를 사용하지 않습니다.

아래 조건을 만족하는 내러티브만 매수 후보 점수 계산에 반영합니다.

```
direction == "rising"
strength > 60
```

즉, 단순히 많이 언급되는 테마가 아니라

- *강도가 충분히 높고, 현재 더 강해지는 중인 테마만 사용합니다.

---

# 5. 입력 2 — theme_map

`theme_map`은 테마와 종목을 연결하는 사전입니다.

```
{
  "반도체": [
    {
      "ticker":"005930",
      "name":"삼성전자"
    },
    {
      "ticker":"000660",
      "name":"SK하이닉스"
    }
  ],
  "방산": [
    {
      "ticker":"012450",
      "name":"한화에어로스페이스"
    }
  ]
}
```

예를 들어 내러티브 테마가 `"AI 및 반도체 슈퍼사이클 지속"`이고,

이 테마가 `"반도체"` 매핑 키와 연결되면

`삼성전자`, `SK하이닉스`에 점수가 분배됩니다.

---

# 6. 점수 계산 방식

## 6-1. 개별 내러티브 점수

각 내러티브는 아래 공식으로 점수를 계산합니다.

```
nar_score =
    strength
  × (1 + delta / 100)
  × day_weight
  × sentiment_mult
  × acceleration
  + shift_bonus
```

---

## 6-2. 공식 구성 요소

| 요소 | 설명 |
| --- | --- |
| `strength` | 해당 테마의 강도 |
| `delta` | 전일 대비 테마 강도 변화 |
| `day_weight` | 최근 날짜에 더 높은 가중치 부여 |
| `sentiment_mult` | 긍정/부정 센티먼트 보정 |
| `acceleration` | 며칠 연속 rising인지에 따른 가속도 보정 |
| `shift_bonus` | 오늘 새로 등장하거나 강해진 테마에 부여하는 보너스 |

---

## 6-3. 날짜 가중치

최근 날짜일수록 더 중요하게 반영합니다.

| 날짜 | 가중치 |
| --- | --- |
| 오늘 | 1.0 |
| 어제 | 0.6 |
| 그저께 | 0.3 |

```
DAY_WEIGHTS= [1.0,0.6,0.3]
```

---

## 6-4. 센티먼트 배율

내러티브의 분위기가 긍정적일수록 높은 배율을 줍니다.

| Sentiment | 배율 |
| --- | --- |
| bullish | 1.2 |
| positive | 1.2 |
| neutral | 1.0 |
| mixed | 0.8 |
| negative | 0.5 |
| bearish | 0.5 |

```
SENTIMENT= {
"bullish":1.2,
"positive":1.2,
"neutral":1.0,
"mixed":0.8,
"negative":0.5,
"bearish":0.5
}
```

---

## 6-5. 가속도 보정

같은 테마가 며칠 연속으로 `rising`이면 추가 점수를 줍니다.

| 조건 | 배율 |
| --- | --- |
| 3일 연속 rising | 1.25 |
| 2일 연속 rising | 1.10 |
| 그 외 | 1.00 |

```
3일 연속 부각되는 테마는 단발성 뉴스보다 더 높은 점수를 받는다.
```

---

## 6-6. Narrative Shift 보너스

오늘 기준으로 테마 변화가 감지되면 보너스 또는 페널티를 적용합니다.

| Change | 점수 |
| --- | --- |
| new | +15 |
| rising | +5 |
| fading | -10 |
| disappeared | -10 |

```
SHIFT= {
"new":15,
"rising":5,
"fading":-10,
"disappeared":-10
}
```

---

# 7. 테마 점수를 종목에 분배하는 방식

내러티브 점수가 계산되면, 해당 테마와 연결된 종목에 점수를 나눠주는 것이 아니라

**매칭된 모든 종목에 동일한 내러티브 점수를 누적**합니다.

예시:

```
내러티브:
AI 및 반도체 슈퍼사이클 지속

매칭 테마:
반도체

연결 종목:
삼성전자
SK하이닉스

결과:
삼성전자 + nar_score
SK하이닉스 + nar_score
```

한 종목이 여러 테마에 포함되어 있다면 점수는 계속 누적됩니다.

```
삼성전자:
반도체 내러티브 점수
+ AI 내러티브 점수
+ 수출 회복 내러티브 점수
= 최종 stock_score
```

---

# 8. 최종 종목 점수

원본 구조에서는 아래와 같은 최종 보정식을 사용합니다.

```
composite =
    stock_score
  × regime_mult
  × ci_modifier
  × risk_adj
  × export_adj
  × confirmation
```

다만 standalone 버전에서는 프로젝트 고유 보정을 제외하고

`EXTRA_MULT = 1.0`으로 처리합니다.

---

## 선택 보정 요소

| 요소 | 설명 | 이식 시 처리 |
| --- | --- | --- |
| `regime_mult` | 시장 국면 보정 | 선택 |
| `ci_modifier` | 내부 Brain/CI 보정 | 생략 가능 |
| `risk_adj` | 리스크 보정 | 생략 가능 |
| `export_adj` | 수출 관련 보정 | 생략 가능 |
| `confirmation` | 추가 확인 시그널 | 생략 가능 |

---

## 시장 국면 배율 예시

| 시장 국면 | 배율 |
| --- | --- |
| risk_on | 1.10 |
| neutral | 1.00 |
| risk_off | 0.85 |
| crisis | 0.70 |

최종 점수는 마지막에 `0~100` 범위로 제한합니다.

```
composite_score = clamp(score, 0, 100)
```

## Freshness Guard

최신 내러티브 날짜가 오늘 날짜가 아니면 빈 결과를 반환합니다.

```
최신 narrative date != today
→ stale 데이터로 판단
→ 빈 결과 반환
```

이 장치는 오래된 뉴스 내러티브가 오늘의 종목 시그널로 잘못 사용되는 것을 막기 위한 안전장치입니다.

# 12. 튜닝 포인트

## 12-1. 최근성 가중치

단타 성향이 강하면 최근 데이터 비중을 더 높일 수 있습니다.

```
DAY_WEIGHTS= [1.0,0.4,0.1]
```

기본값은 아래와 같습니다.

```
DAY_WEIGHTS= [1.0,0.6,0.3]
```

---

## 12-2. 최소 강도 기준

기본값은 `60`입니다.

```
STRENGTH_MIN=60
```

더 보수적으로 운영하려면 `70~80` 수준으로 올릴 수 있습니다.

```
STRENGTH_MIN=75
```

---

## 12-3. 센티먼트 배율

기본 구조는 긍정적 내러티브에 높은 점수를 줍니다.

```
"bullish":1.2
"positive":1.2
"neutral":1.0
"mixed":0.8
"negative":0.5
"bearish":0.5
```

역발상 전략으로 바꾸고 싶다면 negative/bearish의 배율을 높이는 식으로 변형할 수 있습니다.

---

## 12-4. 가속도 기준

현재는 2일 또는 3일 연속 rising 여부를 사용합니다.

| 연속 rising 기간 | 현재 배율 |
| --- | --- |
| 2일 | 1.10 |
| 3일 이상 | 1.25 |

테마 지속성을 더 중요하게 보고 싶다면 3일 연속 조건의 배율을 높일 수 있습니다.

---

## 12-5. Fuzzy Matching 기준

내러티브 테마명과 매핑 테마명이 완전히 같지 않아도 유사도로 매칭합니다.

```
AI 및 반도체 슈퍼사이클 지속
≈ 반도체
```

기본 threshold는 대략 `0.4~0.5` 수준입니다.

| threshold | 성격 |
| --- | --- |
| 낮음 | 더 많이 매칭되지만 오매칭 가능성 증가 |
| 높음 | 더 정확하지만 매칭 누락 가능성 증가 |

# 13. 한계와 주의사항

## 13-1. 입력 품질에 크게 의존

이 전략은 LLM이 생성한 내러티브 JSON을 기반으로 작동합니다.

따라서 입력 내러티브가 부정확하면 종목 시그널도 부정확해질 수 있습니다.

```
잘못된 뉴스 요약
→ 잘못된 테마 판단
→ 잘못된 종목 점수
```

---

## 13-2. 테마 매핑 누락 문제

테마 매핑 사전에 없는 테마는 종목으로 분배되지 않습니다.

예를 들어 `"온디바이스 AI"` 내러티브가 강하게 등장했더라도

`theme_map.json`에 관련 테마와 종목이 없다면 점수화되지 않습니다.

따라서 이 전략은 **theme_map 관리 품질**이 중요합니다.

---

## 13-3. 점수는 수익률 예측값이 아님

`composite_score`는 기대 수익률이 아니라 **상대 순위용 점수**입니다.

```
점수 90점 = 수익률 90% 예상
```

이런 의미가 아닙니다.

```
점수 90점 = 현재 내러티브 기준으로 상대적으로 강한 후보
```

에 가깝습니다.

---

## 13-4. 백테스트는 필수

실제 성과는 반드시 백테스트로 검증해야 합니다.

사용자 제공 기준으로, ascii-prod 주간 리포트에서는 narrative_momentum 평균 d5가 `+2.13%`였지만

알파는 `-1.26%p`로 시장을 따라가지 못한 구간도 존재했습니다.

즉, 내러티브 점수가 높다고 항상 시장 대비 초과수익이 발생하는 것은 아닙니다.

---

# 14. 요약

```
Narrative Momentum 전략은
뉴스에서 강해지는 테마를 찾고,
그 테마에 연결된 종목에 점수를 부여해,
단기 모멘텀 후보를 발굴하는 전략이다.
```

핵심은 아래 4가지입니다.

| 핵심 요소 | 설명 |
| --- | --- |
| 강도 | 테마가 얼마나 강하게 언급되는가 |
| 방향 | 테마가 rising 상태인가 |
| 가속도 | 며칠 연속 강해지고 있는가 |
| 매핑 | 해당 테마가 어떤 종목과 연결되는가 |

# 부록 C. 운영 팁

| 상황 | 권장 처리 |
| --- | --- |
| 매칭되는 종목이 너무 많음 | theme_map을 더 세분화 |
| 점수가 너무 쉽게 100점에 도달 | `STRENGTH_MIN` 상향 또는 최종 정규화 추가 |
| 이상한 테마가 종목에 연결됨 | fuzzy threshold 상향 |
| 특정 테마가 계속 누락됨 | alias 사전 또는 theme_map 키 추가 |
| LLM 결과가 흔들림 | 프롬프트 고정, 뉴스 소스 제한, 출력 스키마 검증 |
| stale 데이터가 섞임 | 최신 날짜 체크 필수 적용 |
