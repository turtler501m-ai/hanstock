# S3.폴더별상세분석

이 문서는 **Hanstock** 트레이딩 플랫폼의 모든 디렉터리와 개별 소스 코드 파일의 기능, 아키텍처 상의 역할, 그리고 상호 관계를 구조적으로 분석한 상세 가이드입니다. 

---

## 1. 애플리케이션 핵심 엔진 (`src/`)

프로젝트 핵심 비즈니스 로직과 다중 실행 엔트리포인트(Entrypoint)가 위치한 최상위 소스 폴더입니다.

### 📂 `src/` (루트)

*   **`trader.py`**: **국내주식 자동매매 총괄 엔진**
    *   **역할**: KIS API 클라이언트(`KIStockAPI`)를 초기화하여 보유 잔고 및 종목 정보를 조회하고, 세븐 스플릿 전략을 기반으로 매수 후보 검색, 계획 수립, 주문 실행을 조율합니다.
    *   **안전망**: `DRY_RUN`, `TRADING_ENV`, `ENABLE_LIVE_TRADING`, `REQUIRE_APPROVAL` 변수를 바탕으로 실제 주문과 시뮬레이션 매매를 철저히 분류합니다. 3% 이상의 일일 손실이 발생하면 자동으로 매매를 멈추는 서킷 브레이커(Daily Loss Halt) 장치도 포함합니다.
*   **`dashboard.py`**: **FastAPI 대시보드 및 백엔드 서버**
    *   **역할**: 주식/선물 잔고 상태, 기술 분석 후보 목록, 실시간 신호 및 실행 계획, 승인 대기열을 제공하는 웹 서버입니다.
    *   **성능 최적화**: KIS API 과다 호출에 따른 트래픽 제한(Rate Limit)을 우회하기 위해 계좌 잔고는 30초, 매수 후보 스캔 데이터는 180초 동안 스냅샷(`.runtime/balance_snapshot.json` 등) 파일로 자동 캐싱합니다.
    *   **환경설정**: `.env` 파일의 변수들을 직접 읽고 쓸 수 있는 설정 관리 API 및 양방향 안전 스위치(Auto Approval, 마스킹 등)를 탑재했습니다.
*   **`config.py`**: **Pydantic Settings 기반의 환경 제어 센터**
    *   **역할**: `.env` 파일과 시스템 환경변수에서 설정값을 타입 세이프(Type-safe)하게 바인딩하여 플랫폼 전체 모듈에 공급합니다.
    *   **주요 변수**: KIS/Telegram/Slack 인증 정보, 세븐 스플릿 매매 파라미터(RSI 임계값, Stop-Loss), 리스크 관리 자본금 설정 등.
*   **`kis_client.py`**: **KIS API 하위 통신 클라이언트**
    *   **역할**: KIS Open API와 HTTP 프로토콜 통신을 처리하는 어댑터 레이어입니다. 토큰 수수, 해시키(Hashkey) 생성, 요청 헤더 구성 등의 공통 로직을 담고 있으며, API 연속 오류 시 요청을 일시 중단하는 서킷 브레이커 패턴이 구현되어 있습니다.
*   **`scheduler.py`**: **주기적 매매 실행 컨트롤러**
    *   **역할**: 국내주식 개장 시간 및 시그널 수집 주기에 맞춰 자동매매 봇(`trader.py`)과 Telegram 수집기(`poll.py`)를 안전하게 스케줄링하여 백그라운드 태스크로 구동합니다.
*   **`execution_plan.py` / `execution_service.py` / `approval_service.py`**: **주문 승인 및 실행 추상화 레이어**
    *   **역할**: 전략 신호와 발굴 후보군 데이터를 대시보드가 이해할 수 있는 균일한 표준 테이블 포맷(`PlanRow`)으로 변환하고, 승인 대기열(`approvals` DB)에 적재하거나 Route를 통해 집행을 대행합니다.
*   **`notifications.py`**: **Slack 채널 메시지 허브**
    *   **역할**: 주식 세션 시작/종료 통계, 발굴된 매수 후보 종목 리스트, 매매 집행 내역 및 시스템 크리티컬 오류 알림을 정제된 마크다운 카드 레이아웃으로 변환하여 송신합니다.
*   **`repositories.py`**: **주식 매매 내역 및 통계 조회 데이터 인터페이스**
    *   **역할**: 데이터베이스 통계 쿼리를 래핑하여 대시보드 화면(보유 자산 비중, 승인 내역, 월별 성과 추이 등)이 복잡한 SQL 작성 없이 정보를 렌더링할 수 있도록 돕습니다.
*   **`rl_env.py` / `train_rl.py` / `train_lgbm.py`**: **강화학습 및 머신러닝 학습 환경**
    *   **역할**: 주식 투자 비중 조절용 PPO 모델 학습을 위한 OpenAI Gym 스타일의 강화학습 환경(`StockTradingEnv`)과 LGBM 분류 모델용 특징(Feature) 생성 및 모델 학습 파이프라인입니다.

## 2. API 클라이언트 연동 모듈 (`src/api/`)

국내외 거래소 및 외부 플랫폼과의 데이터 연동을 전담하는 API 서브 레이어입니다.

### 📂 `src/api/`

*   **`kis_api.py`**: **KIS 국내주식 API 저수준 네트워크 클라이언트**
    *   **역할**: 국내주식 토큰 캐싱 관리, 잔고 조회, 실시간 시세 및 일봉 조회, 주식 매수/매도 현금 주문 등 저수준의 네트워크 통신 모듈입니다.
*   **`kis_futures_api.py`**: **KIS 해외선물 API 저수준 네트워크 클라이언트**
    *   **역할**: 해외선물 잔고 조회, 신규 포지션 진입 및 청산 주문, 미체결 취소 및 시세 조회를 담당하며 모의/실계좌 구분을 완벽하게 처리합니다.
*   **`quantconnect_api.py`**: **QuantConnect 클라우드 제어 API 클라이언트**
    *   **역할**: 미국 퀀트 투자 클라우드 플랫폼인 QuantConnect API를 호출하여 MNQ 선물 매매용 live-paper 프로젝트 컴파일, 배포 개시, 상태 조회, 수동 주문 처리를 래핑합니다.
*   **`bybit_api.py`**: **Bybit 가상자산 API 클라이언트**
    *   **역할**: 루트 디렉터리에 흩어져 있던 독립 바이비트 매매 스크립트를 모듈화하여 `src/api/` 아래로 수용한 구조입니다. 가상자산 잔고 및 호가 조회, 시장가 매매 인터페이스를 갖추고 있습니다.

## 3. 정량적 매매 전략 및 지표 모듈 (`src/strategy/`)

시장 데이터 가공, 수학적 보조지표 추출 및 AI 비중 최적화 알고리즘이 탑재된 플랫폼의 두뇌입니다.

### 📂 `src/strategy/`

*   **`seven_split.py`**: **세븐 스플릿 총괄 전략 매니저**
    *   **역할**: 주식 포트폴리오 스캔, `yfinance` 배치 다운로드 조율, 매매 시그널 결합 및 AI 가중치 분배 로직을 통합하는 전략의 중심지입니다.
    *   **동적 스캔**: KIS 거래량 API가 성공하면 거래량 상위 50종목을 가져오며, 실패할 경우 KOSPI 핵심 67종목 정적 풀(`KOSPI_UNIVERSE`)과 `WATCHLIST`를 교차 검증해 동적으로 타겟 목록을 재구축합니다.
*   **`allocator.py`**: **포트폴리오 자본 할당기**
    *   **역할**: 보유 포지션 수량과 현재 가용 예수금을 비교 분석하여, 최대 보유 슬롯 수(`max_positions`) 안에서 신규 진입할 종목들의 최종 주문 수량과 투자 규모를 리스크 가이드라인에 부합하게 계산합니다.
*   **`indicators.py`**: **수학적 보조지표 엔진**
    *   **역할**: Pandas 등 외부 종속 라이브러리 없이 순수 파이썬 산술 연산을 통해 이동평균(SMA 20/60/120), 볼린저 밴드(Lower/Mid/Upper Band), 상대강도지수(RSI 2/14), MACD(Oscillator 및 골든/데드크로스 판단)를 초고속으로 계산합니다.
*   **`features.py` / `labels.py`**: **머신러닝 데이터 가공 모듈**
    *   **역할**: 주가 및 거래량 시계열을 결합하여 AI 모델 추론용 정규화 특징 패널(Feature Matrix)을 조립하고 매수 적합 여부를 지도 학습용 0/1 라벨로 마킹합니다.
*   **`predict.py`**: **OpenAI ML 후보군 스코어러**
    *   **역할**: 1차적인 보조지표 스코어링을 거쳐 선정된 상위 후보군(`ai_candidate_limit` 이내)의 기술 지표 특징을 OpenAI 프롬프트 기반 API(GPT-5-mini)로 재평가하여 룰셋 점수와 가중 합산하는 기계학습 보조 엔진입니다.
*   **`risk.py`**: **실시간 리스크 엔진**
    *   **역할**: 총 자산 대비 단일 종목의 비중 제한 스키마(`max_single_weight`), 현금 보유 의무 가이드라인(`cash_buffer`), 일일 최대 손실 가능 금액 한계선을 점검하여 신규 진입 가능 여부를 승인/반려합니다.
*   **`router.py`**: **주문 실행 중개기(Order Router)**
    *   **역할**: 대시보드나 백엔드 엔진이 생성한 최종 매매 신호를 바탕으로 실제 KIS 거래 API로의 전송을 최종 라우팅합니다. 이 경로에서 모의/실전 여부와 `DRY_RUN`이 한 단계 더 검증됩니다.

## 4. 데이터베이스 및 유틸리티 공통 모듈 (`src/db/` & `src/notifier/` & `src/utils/`)

안정적인 정보 지속 보존(Persistence)과 공통 유틸리티 클래스가 내장된 기능 레이어입니다.

### 📂 `src/db/`

*   **`repository.py`**: **이종(Dual) 데이터베이스 래퍼 레이어**
    *   **역할**: 프로젝트의 데이터 보존(Persistence)을 전담합니다. 
    *   **유연성**: 로컬 개발 및 간단한 검증 시에는 경량 파일 기반인 **SQLite**(`.runtime/trades.sqlite`)를 고속 바인딩(PRAGMA journal_mode=MEMORY)하며, 클라우드 VM이나 운영 환경에 배포되어 `DATABASE_URL` 환경변수가 주어지면 자동으로 **PostgreSQL** 드라이버(`psycopg2`)로 전환되어 구동됩니다.
    *   **테이블 구성**: `trades`(실제/시뮬레이션 거래 주문 내역, 체결 단가 및 거래소 주문 번호 수록), `decision_logs`(봇이 매매 결정을 내린 시점의 RSI, MACD 등 지표 기록), `approvals`(사용자 수동 확인 대기 대기열).

### 📂 `src/notifier/`

*   **`slack.py`**: **Slack 연동 API 엔드포인트**
    *   **역할**: Slack 웹훅(`SLACK_WEBHOOK_URL`) 구조와 직접 연결되는 단방향 통신 도구로, 플랫폼의 상태 보고와 에러 리포팅 카드를 Slack 채널에 전송합니다.

### 📂 `src/utils/`

*   **`logger.py`**: **글로벌 정제 로깅 도구**
    *   **역할**: 플랫폼 전체에서 동작하는 로그 출력을 관리합니다. 터미널 표준 출력과 `logs/` 디렉터리 하위의 텍스트 로그 파일로 로그를 동시에 저장하며, KST 한국 표준시 기준으로 포맷팅된 타임스탬프를 보증합니다.

## 5. 해외선물 시그널 시스템 (`src/futures_signals/`)

Telegram 해외선물 시그널의 수집, 파싱, 정규화, 검증 및 KIS 해외선물 대시보드 주문 집행을 묶는 자동화 파이프라인입니다.

### 📂 `src/futures_signals/`

*   **`poll.py` / `telegram_collector.py`**: **Telegram 채널 메시지 폴러**
    *   **역할**: Telethon 비동기 라이브러리를 활용하여 설정된 Telegram 해외선물 채널 정보(`TELEGRAM_TARGET_CHANNELS`)로부터 실시간 텍스트 메시지를 가져옵니다.
*   **`parser.py`**: **시그널 정규화 정밀 파서**
    *   **역할**: 수집된 대화 텍스트에서 매매 방향(매수/매도/롱/숏), 선물 품목명(나스닥, 골드, 금 등 한국어 매핑 포함), 진입 가격 및 목표 가격대(손절가 SL, 익절가 TP)를 정규표현식(Regex)을 이용해 깔끔한 구조체 객체(`FuturesSignal`)로 해체 및 파싱합니다.
*   **`verifier.py`**: **가격 타당성 검증기**
    *   **역할**: 파싱 완료된 시그널의 매매 방향과 지정가들의 모순 여부(예: 롱 포지션에서 손절 타겟이 진입 가격보다 높게 적혀 있는 논리적 설정 에러)를 필터링하여 불량 신호를 사전에 차단합니다.
*   **`executor.py`**: **선물 주문 처행기**
    *   **역할**: 저장된 시그널 상태와 KIS 해외선물 거래 API를 연결하여 대시보드 화면상에서 원클릭으로 모의/실제 선물 주문을 시장에 전송할 수 있는 실행 장치입니다.
*   **`db.py` / `repository.py` / `models.py` / `service.py`**: **해외선물 전용 데이터 영속 레이어 및 서비스 패키지**
    *   **역할**: 해외선물 메시지의 원문 및 파싱 정규화 상태를 수록하는 별도의 DB(`.runtime/signals.db`) 핸들러 및 데이터 파이프라인 흐름을 관장하는 추상 비즈니스 메서드 세트입니다.

## 6. 외부 플랫폼 이종 알고리즘 연동 (`src/integrations/`)

외부 트레이딩 플랫폼에서 구동될 소스 및 플랫폼 보조 컴포넌트가 위치하는 하이브리드 영역입니다.

### 📂 `src/integrations/quantconnect/mnq_paper_auto/`

*   **`main.py`**: **QuantConnect 전용 주식/선물 매매 알고리즘**
    *   **역할**: QuantConnect 라이브 트레이딩 플랫폼 클라우드 위에서 독립 실행되는 파이썬 기반 거래 알고리즘입니다. 마이크로 나스닥 E-mini 선물 지표를 관측하며 대시보드와 API 신호로 소통하고 최대 계약 한도(`MAX_CONTRACTS=3`) 등 안전 규칙을 자율 집행합니다.
*   **`config.json`**: **QuantConnect 프로젝트 환경 설정 파일**
    *   **역할**: 클라우드 컴파일 및 배포 노드 지정용 고유 메타데이터 파일입니다.

## 7. 프론트엔드 인터페이스 (`web/`)

Jinja2 템플릿과 CSS, Vanilla JS 기반의 반응형 반응형 화면 레이아웃입니다.

*   **`web/templates/`**: **Jinja2 기반 HTML 뷰**
    *   **`index.html`**: 보유 잔고, 현금 비중 도넛 차트, 성과 차트 및 **주식 자동매매 승인 대기열**을 출력하는 종합 대시보드 첫 화면입니다.
    *   **`futures_signals.html`**: Telegram 수집 상태 및 정규화 시그널, **해외선물 원클릭 거래 콘솔**을 탑재한 해외선물 메인 관리창입니다.
    *   **`ai_dashboard.html` / `finrl.html`**: AI 모델 스코어 추세, 머신러닝 스캐닝 성과 통계 및 FinRL/PyPortfolioOpt 최적화 비중 시각화 보조 화면입니다.
    *   **`env_settings.html`**: `.env` 환경 변수들의 양방향 수정/반영을 지원하는 대시보드 설정 페이지입니다.
    *   **`fragments/`**: QuantConnect 탭(`futures_tab_quantconnect.html`), 주문/체결 탭, 실시간 시그널 등 화면 일부분만 비동기로 변경해 주는 모듈화 HTML 조각 모음입니다.
*   **`web/static/`**: **CSS 및 JavaScript 자원**
    *   Vanilla CSS 기반의 다크 모드/글래스모피즘 계열 현대적 스타일시트와, Axios 비동기 AJAX 처리, Chart.js 그래프 바인딩 로직을 포함하고 있습니다.

## 8. 로컬/VM 실행 및 배포 스크립트 (`scripts/`)

플랫폼의 이중화 구동(Windows 로컬 개발 환경 및 Linux VM 운영 환경)을 원클릭으로 안전하게 동기화해 주는 쉘 스크립트 모음입니다.

*   **`scripts/local/`**: **로컬 Windows 환경 제어**
    *   `server.cmd`: 로컬에서 대시보드 백그라운드 서버를 재시작/정지/로그 조회하는 바로가기 배치 파일입니다.
    *   `deploy-vm.ps1`: 로컬에서 개발 및 테스트가 완료된 소스를 원격 저장소에 반영한 뒤, GCP VM 호스트(`hanstock-server5`)에 접속하여 `git pull` 및 서버 재시작까지 자동으로 원격 대행해 주는 파워쉘 배포 파이프라인 스크립트입니다.
    *   `connect-vm.ps1`: GCP VM 서버로의 SSH 보안 터널링 연결을 빠르게 열어 주는 유틸리티입니다.
    *   `vm-dashboard.ps1`: VM의 리소스 사용 상태와 포트 바인딩 상황을 로컬에서 원격으로 진단합니다.
*   **`scripts/vm/`**: **원격 Linux 운영 서버 환경 제어**
    *   `server.sh`: VM 리눅스 환경 상에서 Uvicorn 데몬 웹 대시보드 프로세스를 백그라운드로 안전하게 부팅하고 에러 출력을 `.runtime/dashboard-server.err.log`에 실시간 기록/추적하는 가디언 쉘 스크립트입니다.
    *   `daily-auto.sh`: 매일 아침 주식 시장 개장 전 크론탭(Cron) 스케줄에 의해 가동되어 주식 잔고 상태 스캔 및 자동매매 엔진을 기동하는 스크립트입니다.
    *   `update.sh`: 원격 GitHub 저장소로부터 최신 `main` 소스코드를 가져와 안전하게 로컬 환경에 덮어쓰고 pip 의존성을 리로드합니다.

## 9. 로컬 점검 및 무결성 도구 (`tools/`)

커밋 전 로컬 테스트 및 DB 진단을 지원하는 디버깅 유틸리티 풀입니다.

*   **`verify-local.ps1`**: **로컬 소스 무결성 파이프라인**
    *   **역할**: 로컬 코드를 원격 저장소에 푸시(Push)하기 전에 에디터 포맷 점검, 한글 인코딩 깨짐 유무 점검 및 전체 유닛 테스트 수트를 일괄 수행하여 오작동 없는 안전한 상태임을 증명해 주는 사전 커밋 점검 툴입니다.
*   **`demo-trading-rehearsal.py`**: **주식 매매 가상 시뮬레이션 연습기**
    *   **역할**: KIS Open API 접속 없이 로컬 환경의 Mock 데이터를 결합하여 세븐 스플릿 매매 시나리오를 고속 리허설하여 persistence 저장소 동작을 시험합니다.
*   **`telegram-login.py`**: **Telegram Telethon 세션 인증 도구**
    *   **역할**: 해외선물 시그널 수집기 최초 구동 시 2단계 인증 코드 및 모바일 세션 토큰 파일(`.runtime/futures_telegram.session`)을 로컬에서 안전하게 생성해 줍니다.
*   **`check-encoding.ps1`**: **인코딩 무결성 감사 툴**
    *   **역할**: 소스코드 전체를 전수 조사하여 Windows의 ANSI/CP949 코드로 작성되어 Git 상에서 깨질 수 있는 파일들을 UTF-8 LF 형식으로 자동 감지하여 수정합니다.
*   **`check-kis-account.py`**: KIS Open API 인증 및 계좌 상태의 실시간 정상 접근 가능 여부를 독립 점검합니다.
